是否计提：标识该行在当期正式计提/发放链路中是否参与入账；否时仍可保留试算金额供核对（如映射停发、区间外仅展示等）。与「提成计算区间」「发放状态」联动规则以实现为准。

· 毛利调整结转（Q8）：上月末滚入台账，在基数维护 / 基数方案公式中引用（如 #{上月结转_毛利调整}）；与列表列「计算净利润（毛利−表内毛利扣）」分列，阶梯/比例计提基数 = 计算净利润 + 该结转项（示例：31335+(−200)=31135）。本期结转明细以表外成本 / 月度调整入账为准。
· 提成调整结转：上月末滚入台账，由月度结转/提成导入进入引擎，在规则层扣减链引用 #{上月结转_提成调整}，进入扣减/调整或与实发提成合并；勿与基数公式重复扣减。
两列均可为负；是否与当月公式合并及合并顺序以引擎配置为准。

直客：计提基数 = 当月归因该员工的计算净利润合计（阶梯落在净利润上）。「表外提成扣」不进净利润，在「扣减/调整」体现。
同行：计提基数 = 计费重合计 × 固定提成点；毛利/净利润列保留便于横向对账，不参与本条计提乘积。
客服：计提基数 = 应收汇总（可按产品约定扣减免赔等后进「净应收」，以实现为准）；成本/毛利列可为空。
渠道：计提基数 = 对内毛利（示意：毛利 − 渠道返点 − 毛利性质导入扣减）；与场景矩阵 CHANNEL_MANAGER_SETTLE / COMMISSION_GP_CHANNEL 对齐。

二期内陆费 / 操作费：与运单底表、月度归档口径一致；金额可汇总展示，单号明细通过「单号明细」打开弹窗（运单号、件数、计费重、扣减费用，示意）。个税列高亮为财务关注字段；毛利扣除（重算工资）为影响应发前的工资联动扣项（示意）。应发提成 / 实发提成命名与发放台账一致；工具条支持勾选 → 特殊放行（必填放行原因）、提成调整并写日志。确认发放：若存在是否计提=否且未完成行级放行+原因，须填写批量发放说明（二选一，见弹窗）。

发放与导入分工（原型约定）：确认发放提成走批量勾选 → 生成发放批次（审计友好、与试算快照对齐）。 万能池 / 绩效系数等属于事前数据校正，通过「月度调整表」导入入账后再试算；不宜用「导入」代替「发放」动作。

发放回款核验与未核销明细：列表中的「已达下限 / 临近阈值 / 未达下限」仅对照配置的回款率下限做可视化提示，不等于系统自动允许或禁止放款；是否与计提、捆绑规则、暂缓名单等联动以实现为准。点开「未回款明细」可核对未核销账单费用，便于财务结合账期与客户约定人工判断是否纳入本轮发放。
