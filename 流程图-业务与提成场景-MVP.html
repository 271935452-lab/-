提成试算：命中顺序（账期一致）

与 PRD 摘要 16、试算说明一致：三者对同一账期同时有效才参与计算。

flowchart TD
  Z[运单底表 + 账期] --> M{人员映射命中?}
  M -->|否| X[不参与计提 / 需人工]
  M -->|是| P{规则包命中?}
  P -->|否| X
  P -->|是| R{规则命中?}
  R -->|否| X
  R -->|是| OK[进入基数与公式计算]

命中顺序：人员映射 → 规则包 → 规则；映射需先对齐分公司、提成角色、员工状态、映射周期与账期。

提成引擎：Step2～Step5 抽象流水线

对应域名中规则包命中后的基数 / 公式 / 结果层处理。

flowchart LR
  S2[Step2 规则包命中] --> S3[Step3 基数汇总]
  S3 --> S4[Step4 公式计发]
  S4 --> S5[Step5 结果校验扣减]
  S5 --> N[封顶/负毛利/上月结转/提成扣等]

基数可含：应收合计、有效应收、净利润相关毛利与表外扣减等，以基数方案配置为准。

场景 1 / 4：直客业务员 · 净利润固定阶梯

直客业务员固定提成点与「净利润阶梯」共用骨架（域名 四、1 与 4）。

flowchart TD
  S1["Step1 运单底表 + 人员映射主档对齐"] --> S2["Step2 命中规则 入职时段等"]
  S2 --> S3{"Step3 包内是否优先客服"}
  S3 -->|是| CS["先算客服 基数乘点减提成扣"]
  S3 -->|否| DS
  CS --> DS
  DS["Step4 直客业务员 净利润乘阶梯点 减客服提成扣 减表外提成扣"] --> OUT1[直客实发提成]

净利润口径示意：(毛利 − 毛利扣) 或包内定义的净利润；客服优先时基数常为「毛利 − 表外」等，以规则包为准。

场景 2：直客业务员 · 净利润阶梯 + 累加点

flowchart TD

场景 3：总部直客主管 · 组员应收 × 提成点 ×（1−绩效%）

域名 四、3；组员维度需先确认主管下属映射。

flowchart TD

人员映射：修改截止时间引发区间拆分/合并（示意）

抽象自域名「人员映射修改原则」与 A/B/C 区间示例。

flowchart TD
  START[修改某条映射截止时间] --> Q{记录是否已生效或已参与试算?}
  Q -->|否| U1[可直接更新原记录或按未生效规则处理]
  Q -->|是| U2[不允许覆盖: 生成新映射段]
  U2 --> SPLIT{与相邻区间关系}
  SPLIT --> A[拆段: 前段保留 / 中段新值 / 后段另条]
  SPLIT --> B[合并为一条连续区间]
  SPLIT --> C[多段并存: 按生效区间唯一命中]

具体拆段规则以《人员映射表》PRD 与引擎校验为准；冲突时置空或报错。
