# 编辑规则页 PRD

## 1. 文档信息

- **文档名称**：提成规则「编辑规则」页 PRD（MVP）
- **关联原型**：`编辑规则-MVP.html`
- **关联模块**：规则包 / 基数方案 / 人员映射（账号与角色）/ 运单底表 / 提成试算
- **面向对象**：产品、前端、后端、测试、规则引擎
- **当前版本**：V1.0（与 MVP 原型对齐；引擎落库字段名以实现为准）

---

## 2. 目标说明

编辑规则页用于在单条规则维度维护：**命中条件**（谁在什么业务数据下适用本规则）、**计提与扣减链路**（基数、阶梯/固定比例、提成扣类型）、**版本与生效期**。  

本页需与试算、归档结果可对账：命中过程可解释、可审计，避免出现「页面上有字段但引擎不知道是否参与匹配」的灰区。

---

## 3. 功能范围

### 3.1 本期（MVP）范围

- 规则基本信息、生效区间、状态、说明文案。
- 指定客户、客户匹配方式、客户范围。
- 提成扣配置（总开关 + 类型多选 + 说明）。
- 销售产品与结算相关参数（含「同行业务员」与相加组说明）。
- 适用角色与匹配控制（分公司、小组、提成角色、`service_mode`、优先级、入职区间、员工状态）。
- 条件表达式 / 计算公式编辑器（占位符插入、模板、撤销）。
- 基数方案引用及计算方式（毛利阶梯 / 计费重阶梯 / 固定比例）配置。
- 保存草稿、保存并发布、试算预览与变更日志（原型按钮；联调细节可二期补全）。

### 3.2 不在本期范围

- 分段函数型多段折线（原型提示二期对齐表达式后开放）。
- 封顶策略与试算的深度联动（部分选项为二期占位）。
- 完整审批流、多人协同锁单。

---

## 4. 页面入口与布局

- **入口示例**：从规则引擎列表进入「编辑」；或从规则包维护跳转至指定规则编码。
- **布局**：左侧主导航（提成管理模块）+ 本页内「模块导航」锚点（基本信息、提成扣、销售产品、匹配控制、指定客户、表达式、基数、备注与审计）。
- **底部操作条**：取消、保存草稿、保存并发布（与顶栏按钮语义一致时可二选一展示，以最终实现为准）。

---

## 5. 命中逻辑总原则（引擎侧）

1. **时间窗**：运单/账期落在「生效起始～生效截止（空=长期）」且规则状态为可参与计算时，才进入候选集。
2. **结构化条件与表达式**：MVP 要求 **结构化字段（分公司、小组、角色、`service_mode`、入职区间、员工状态、销售产品、客户范围等）与「条件表达式」语义一致、可校验**；保存时建议由系统根据结构化选项生成/合并表达式，或校验用户手写表达式不与之冲突（具体策略由开发定，产品要求：**试算命中原因可读到一致维度**）。
3. **多规则冲突**：同一互斥组内依赖优先级与互斥策略（与规则包配置一致）；「相加组」内多条规则按产品维度交集分别命中后相加（见销售产品章节）。
4. **账号维度**：`#{来源业务账号ID}` 等应在表达式或映射结果中可落到唯一 ID，明细可回显；**个别累计提成点**场景下用于 **白名单收窄**，见 **§7.5**。

## 6. 字段是否参与「运单/员工匹配」总表

说明：**「参与匹配」**指影响「本条规则是否进入该运单/该员工的计提链路」；**「不参与匹配」**指仅影响展示、计算口径或落库元数据，不单独作为运单过滤条件。

| 模块 | 字段 | 是否参与运单/员工匹配 | 备注 |
|------|------|------------------------|------|
| 基本信息 | 规则名称 | 否 | 展示与检索。 |
| 基本信息 | 规则编码 | 否 | 唯一标识；创建后只读。 |
| 基本信息 | 所属规则包 | 间接 | 不直接过滤运单；影响包内编排（互斥/相加）与默认结算方式继承。 |
| 基本信息 | 状态 | 是（门控） | 非「可计算」状态则整体不参与命中。 |
| 基本信息 | 生效起始 / 生效截止 | 是 | 时间窗过滤。 |
| 基本信息 | 规则说明 | 否 | 对内说明。 |
| 指定客户 | 指定客户列表 | 条件参与 | 见 §7.2；与「客户范围」组合决定何时生效。 |
| 指定客户 | 客户匹配方式 | 是 | 决定用名称、编码还是分组做等值/映射匹配。 |
| 指定客户 | 客户范围 | 是 | 全部 / 仅指定 / 排除，决定列表是否构成过滤条件。 |
| 指定客户 | 客户命中说明 | 否 | 文案。 |
| 提成扣 | 提成扣类型多选 | 否（匹配）/ 是（计算） | 不参与「是否命中」；参与命中后的应发计算链。 |
| 提成扣 | 是否启用提成扣 | 否（匹配）/ 是（计算） | 总开关关闭则扣减项整体不执行。 |
| 提成扣 | 提成扣计算说明 | 否 | 口径说明。 |
| 销售产品 | 销售产品（多选） | 是 | 与底表「销售产品」维度对齐；见 §7.3。 |
| 销售产品 | 结算方式 | 否（匹配） | 影响结算节奏，不用于单票是否命中规则。 |
| 销售产品 | 基本提成点（只读） | 否（匹配） | 计算因子。 |
| 销售产品 | 累计规则 / 累计上限 | 否（匹配） | 计算因子。 |
| 匹配控制 | 分公司 / 小组 | 是 | 与人员或运单归属维度对齐（以实现映射为准）。 |
| 匹配控制 | 适用提成角色 | 是 | 至少选一项；与人员映射角色一致才命中。 |
| 匹配控制 | 服务场景（`service_mode`） | 是 | **有/无业务员客服** 见 §7.1。 |
| 匹配控制 | 匹配优先级 | 是（冲突时） | 同组内比较用。 |
| 匹配控制 | 入职时间起止 | 是 | 员工入职日期在区间内（止为空=无上限）。 |
| 匹配控制 | 员工状态 | 是 | 试用/转正/全部。 |
| 表达式 | 条件表达式 | 是 | 应与结构化条件一致；可扩展未表单化的维度。 |
| 表达式 | 计算公式 | 否（匹配） | 仅影响金额计算。 |
| 基数与计算方式 | 基数方案编码 | 否（匹配） | 命中后取基数。 |
| 基数与计算方式 | 阶梯/固定比例等各子字段 | 否（匹配） | 仅影响计提算法与回款门槛等计算策略。 |
| 备注与审计 | 内部备注 / 最近修改 | 否 | 审计展示。 |

## 7. 重点：「有 / 无」维度的匹配规则（缺省与空值）

### 7.1 有无业务员客服（`service_mode`：全部场景 / 有业务员客服 / 无业务员客服）

**数据来源**：运单或映射结果上的服务场景标记（与《人员映射》中客服类角色 `service_mode` 口径一致）。  

| 规则页选项 | 底表/映射为「有业务员客服」 | 底表/映射为「无业务员客服」 | 底表缺失或未知 |
|------------|----------------------------|------------------------------|----------------|
| **全部场景** | 命中 | 命中 | **建议：视为不通过本维度**（仍受其他维度约束）；或统一视为「全部」由数据治理保证非空——**上线前需二选一并写死**。 |
| **有业务员客服** | 命中 | **不命中** | **不命中**（保守策略：未知不按「有」处理）。 |
| **无业务员客服** | **不命中** | 命中 | **不命中**（保守策略：未知不按「无」处理）。 |

**产品建议（MVP）**：底表 `service_mode` 强制枚举非空；若历史数据存在空值，导入校验报错或归入「全部场景」专用规则，避免 silent wrong pay。

---

### 7.2 有无客户 / 客户未关联（指定客户 + 客户范围）

「户」若指 **客户主体**：以运单底表「客户标识」（名称/编码/分组 ID，取决于「客户匹配方式」）是否可取值为核心。

| 客户范围 | 指定客户列表为空 | 指定客户非空 | 运单客户字段为空或未关联 |
|----------|------------------|--------------|---------------------------|
| **全部客户** | 不对客户维度过滤；等价于「有无客户」不影响命中 | 仍应对 **其他维度** 命中；若产品希望「选了客户就必须限制」，**当前原型易歧义**——建议：**列表为空时忽略指定客户区**，仅「仅指定/排除」强制依赖列表 | **命中**：未对客户做排除；若业务上禁止无客户走提成，应通过 **条件表达式** 或独立数据校验规则拦截 |
| **仅指定客户** | **不命中任何运单**（无法构成白名单） | 运单客户与列表按匹配方式命中则 **命中** | **不命中**（无客户可比对） |
| **排除指定客户** | 退化为「不排除任何人」= 等价全部客户（或实现为不命中——需统一） | 运单客户在列表中则 **不命中**；不在列表则 **命中** | **命中**（空值不在排除名单中时，默认不排除） |

**产品建议**：保存「仅指定客户」且列表为空时 **前端强校验拦截**；「排除」且列表为空时提示用户改为「全部客户」或补充名单。

---

### 7.3 有无销售产品（底表销售产品为空 vs 有值）

规则侧为 **销售产品多选集合**（至少一项为 MVP 必填）。

| 规则配置 | 运单销售产品有值且落在规则集合内 | 运单销售产品有值但 **不在** 规则集合 | 运单销售产品 **为空/未打标** |
|----------|-----------------------------------|----------------------------------------|------------------------------|
| 已选产品集合 **非空** | **命中**（产品维度通过） | **不命中** | **不命中**（无法证明属于所选产品；与「仅指定客户」空列表类似，避免误发） |

**同行业务员 + 相加组（与原型说明一致）**：同一员工引用相加组内多条规则时，按 **产品交集** 分别命中并 **相加**；若某运单产品为空，则 **无法与任何依赖产品的规则行求交集**，该运单在相加组内对产品敏感规则贡献为 0（除非另有「不校验产品」的兜底规则，需单独配置并在 PRD 中显性命名）。

**产品建议**：对存在空产品的渠道，配置一条 **明确覆盖「产品为空」的兜底规则**（表达式中显式判断）或推动底表清洗；不依赖隐式默认。

### 7.4 其它常见「无值」维度（简表）

| 维度 | 规则侧「未限制」表现 | 底表/员工侧为空 | MVP 建议 |
|------|----------------------|-----------------|----------|
| 生效截止 | 空 = 长期有效 | — | 合法。 |
| 入职时间止 | 空 = 无上限 | — | 合法。 |
| 员工状态 = 全部 | 不限制试用/转正 | — | 合法。 |
| 小组 = 全部小组 | 不限制小组 | 员工无小组 | **命中**（若与其它维度一致）；若业务禁止，则用表达式约束。 |
| `#{来源业务账号ID}` 在表达式中写死 | — | 底表账号与表达式不等 | **不命中**；表达式未包含该维度则不在该维度过滤。 |

### 7.5 同包个别业务员「累计提成点」（双规则 + 表达式白名单 + 优先级）

**场景**：人员映射命中**同一规则包**，且与大众相同的结构化命中维度（分公司 / 部门小组 / 提成角色 / 映射与账期等）下，仅**少数业务员**需启用「**累计提成点**」（年资累加、上限封顶等），其余人仍用固定点或不累计。

**推荐配置（不拆包）**：在同一规则包内维护 **两条互斥规则**（须在同一**互斥组**内，避免双发）。

| 规则 | 结构化条件 | 条件表达式（系统） | 累计规则 / 上限 | 匹配优先级（示例） |
|------|------------|-------------------|-----------------|-------------------|
| **A 大众默认** | 与现网一致 | 与结构化一致，**不追加** `#{来源业务账号ID}` 白名单 | **不累计** | 较低，如 **100** |
| **B 个别人累计** | 与 A 对齐（分公司/小组/角色/产品等） | 在 A 的条件基础上追加 `&& #{来源业务账号ID} in ['id1','id2',…]`（ID 以人员映射/主数据唯一账号为准） | **启用累计**并配置上限 | **高于 A**，如 **310** |

**命中顺序**：互斥组内 **优先级高者先评估**；`in` 白名单为**更窄**条件，故 B 应先于 A 命中；不在白名单内的员工仅命中 A。

**引擎与审计**：同一员工、同一账期、同一互斥上下文 **仅允许命中 A 或 B 之一**；试算与归档日志须记录最终命中的 **规则编码**。

**保存校验（建议）**：规则 B 的 `in [...]` 不得为空；若 A 与 B **优先级相同**且存在员工同时满足两式，须 **阻断保存** 或按字典序稳定打破（产品需二选一并写死）。

## 8. 分模块字段说明（与原型对齐）

### 8.1 基本信息

- **规则名称**（必填）、**规则编码**（必填、只读）、所属规则包、状态、生效起始（必填）、生效截止（可空）、规则说明。

### 8.2 指定客户

- **指定客户**（多选）、**客户匹配方式**（名称精确 / 编码精确 / 分组）、**客户范围**（全部 / 仅指定 / 排除）、客户命中说明。  
- 匹配规则见 **§7.2**。

### 8.3 提成扣配置

- **提成扣类型**（多选，**MVP 仅两种**）：**客服提成**、**表外导入**；与《域名说明》「三、提成扣说明」一致。**是否启用提成扣**（启用/不启用）、提成扣计算说明。  
- 不参与命中，参与命中后金额链（与计算公式中占位符一致）。

#### 8.3.1 业务口径（《域名说明》「三、提成扣说明」对齐）

1. **客服提成（客服提成扣）**  
   - 当期**所属规则包**若存在需计提/需扣的**客服提成**口径，引擎须**先完成客服侧计提**，得到当期可确认的客服提成结果，再作为 **`#{客服提成扣}`** 参与本条规则及同包内其他主体角色之后的应发扣减链。  
   - 即：**有客服提成则先算客服，再算当前规则主体**；与业务侧「业务员先跑、主管后跑」等顺序原则一并体现在跑批/试算步骤中。  
   - 未命中客服或未配置时：该项按 **0** 或引擎约定默认，须与试算日志步骤一致。

2. **表外导入（表外导入扣）**  
   - 仅汇总**当期**在**表外导入 / 月度调整**中入账、且记录类型为 **「提成扣」** 的表外成本金额，作为 **`#{表外导入扣}`**（系统字段名以实现为准）。  
   - **非「提成扣」类型**的导入**不得**计入本扣减项（可进入毛利扣、基数层表外成本等其他口径，见月度调整表 PRD）。

3. **与本页表单的关系**  
   - 「是否启用提成扣」关闭：本条规则不执行任何提成扣类型扣减（或全部视为 0，以实现为准）。  
   - 开启时：**提成扣类型**仅枚举 **客服提成**、**表外导入** 两项；仅对**勾选**项按 §8.3.1 取数并代入「计算公式」中 `#{客服提成扣}`、`#{表外导入扣}`。

### 8.4 销售产品与结算参数

- **销售产品（规则匹配）**（必填、多选）、结算方式、基本提成点（只读）、累计规则、累计上限。  
- 匹配规则见 **§7.3**；与「相加组」关系见原型绿字说明。

### 8.5 适用角色与匹配控制

- **分公司**、**小组**、**适用提成角色**（必填、至少一项）、**服务场景（`service_mode`）**、匹配优先级、入职起止、员工状态。  
- **服务场景**见 **§7.1**。  
- **匹配优先级**：同互斥组内比较；**§7.5**「累计提成点」双规则场景下，个别人规则须 **高于** 大众规则。  
- **匹配控制区 hint**（原型）：与条件表达式中 `#{来源业务账号ID} in [...]` 白名单配合说明。

### 8.6 条件表达式与计算公式

- **条件表达式**：门槛与命中统一写入；占位符包括：`#{分公司}`、`#{小组}`、`#{提成角色}`、`#{入职时间}`、`#{员工状态}`、`#{来源业务账号ID}`、`#{组员月均票数}`、`#{销售产品}`、`#{指定客户}` 等（以实现为准）。  
  - **累计提成点个别人**：在大众条件后追加 `&& #{来源业务账号ID} in ['id1','id2']`，并与 **§7.5** 的「双规则 + 优先级」一致。  
- **计算公式**：净利润基数、提成点、各扣减项等；不参与是否命中。

### 8.7 基数方案与计算方式

- **基数方案编码**（必填引用）。  
- **毛利阶梯**：阶梯类型、回款发放门槛、分档上下界与比例。  
- **计费重阶梯**：基数类型、单位切换方式、分档与单价。  
- **固定比例**：比例、比例基数口径、回款门槛、封顶策略（占位）、展示精度等。

### 8.8 备注与审计

- 内部备注、最近修改只读展示。

## 9. 校验与保存

- **必填**：规则名称、规则编码、生效起始、销售产品至少一项、分公司、小组、适用提成角色至少一项、基数方案（及当前页签要求的子字段）。  
- **组合校验**：「仅指定客户」且客户列表为空 → 拦截保存；`service_mode` 非「全部」时建议校验底表字段非空策略与 **§7.1** 一致。  
- **保存草稿 vs 发布**：发布后生成新版本；草稿可改。与规则包锁定关系以实现为准（原型文案提示规则包锁定后部分维度只读）。
