编辑规则页 PRD（MVP · 开发测试版）

二期标题前带角标的小节为与问题清单 16～22、《域名说明》第 23～24 条及《07_二期原型待产品确认问题-产品回复》等对齐的补充说明，实现范围以立项为准。

背景与目标

在单条规则上维护「命中条件、计提与扣减链、版本与生效期」，与试算/归档可对账；命中过程可解释，避免「页面上有字段但引擎不参与匹配」的灰区。

二期命中总原则

  • 账期与命中链（16）：单条规则须在自身生效区间内；试算全链路须先通过人员映射、再命中规则包、最后命中本规则，三者对同一账期同时有效方可参与计算。
  • 规则包排他（17）：同一提成角色 + 分公司 + 部门 + 小组在同一时间片内仅允许一个「生效中」规则包，避免同维多套包并行。
  • 复用（22）：基数方案可复用；规则与规则包不可复用（规则编码唯一，复制须新码）。
  • 列表状态与账期（24）：规则主档/规则包列表为「已失效」等运营展示态时，不自动排除「数据账期仍落在该规则（及所属包、人员映射）生效区间内」的补算；引擎以数据账期切片 + 已发布快照判定；计费日志须带规则/包版本标识。若字典定义「已废弃」则禁止全账期。
  • 《07 产品回复》P6～P9（规则模块）：P6 包层保留默认编排默认值；规则层「互斥组」输入 MVP 去掉，默认同角色互斥；多角色包内按优先级顺序。P7 销售产品仅规则层参与匹配；相加=同包同角色同人多规则命中则数值类结果相加；苏州整柜/非整柜优先运单底表「是否整柜」逻辑列。P8 指定客户 MVP 默认白名单 INCLUDE；EXCLUDE/ALL 不暴露、后端枚举预留。P9 #{员工工号} 与人员映射表中的员工工号一致，须进内置变量接口并与试算明细回显对齐。
  • 运单/账期落在生效区间内，且规则状态允许参与计算，才进入候选集。
  • 结构化字段（分公司、小组、角色、服务场景、入职区间、员工状态、销售产品、客户范围等）与「条件表达式」语义应一致或可校验；试算命中原因可读到一致维度。
  • 互斥 / 相加（与《07 产品回复》P6 对齐）：同角色互斥与包内默认编排由规则包主档承载；规则编辑页不再维护「互斥组/相加组」编号输入。同角色多条规则仍按匹配优先级 + 销售产品交集命中；多规则相加由包级「同角色相加」与矩阵定义触发。
  • 工号类维度建议在表达式中使用 #{员工工号} 与 in ['工号1',…] 白名单，取值以人员映射表工号为准，明细可回显。
  • 同包个别「累计提成点」：见下方专节，与「匹配优先级」字段说明一致。

同包个别业务员「累计提成点」（双规则 + 表达式）

当人员映射命中同一规则包、结构化条件（分公司/部门小组/角色/时间）与大众相同，仅少数员工需累计提成点时，不单独拆包；在同一规则包内维护两条规则。

  • 规则 A（大众默认）：结构化条件覆盖全员；条件表达式不写 #{员工工号} 白名单；「累计规则」选不累计；匹配优先级取较低值（如 100）。
  • 规则 B（个别人累计）：复制规则 A，销售产品与指定客户、分公司、小组、角色等与 A 对齐（均在结构化区块维护）；开启累计规则与累计上限；在条件表达式中追加 && #{员工工号} in ['真实工号_1','真实工号_2']（列表以人员映射表工号为准）；匹配优先级设为高于 A（如 310），且与 A 处于同一互斥组，保证「窄规则先命中、否则落默认」。
  • 引擎行为：对同一员工同一账期，仅允许命中 A/B 之一；试算/日志须输出最终命中的规则编码以便审计。
  • 校验：保存时检测 B 的白名单是否为空；检测 A 与 B 是否存在「同优先级且条件可同时为真」的死锁或双命中，必要时阻断保存。

字段是否参与「运单/员工匹配」

「参与匹配」= 影响本条规则是否进入该运单/员工的计提链路；「不参与」= 仅展示、元数据或命中后的计算口径。

  • 不参与匹配：规则名称；规则编码；规则说明；客户命中说明；提成扣类型/总开关/扣减说明（参与命中后金额，不参与是否命中）；基本提成点只读、累计规则/上限；计算公式全文；基数方案及阶梯/固定比例各子字段；内部备注、最近修改。（《07 产品回复》P5：「结算方式」第一版从表单移除。）
  • 参与或门控：状态；生效起始/截止；所属规则包（间接影响包内编排）；指定客户（多选列表 + 客户命中说明，列表数据来自客户表·已开户；本页原型已移除「客户匹配方式」「客户范围」下拉，引擎侧默认名称精确匹配、仅列表命中）；销售产品多选（待选为销售产品主数据·已启用）；分公司、小组、适用提成角色（系统字典，与人员映射表·提成角色同源）、服务场景（service_mode）、匹配优先级、入职起止、员工状态；条件表达式。
  • 条件表达式快捷 token：不提供 #{销售产品}、#{指定客户} 插入按钮；销售产品、指定客户仅由「销售产品与结算参数」「指定客户」结构化区块配置，与表达式并存时以结构化为准（表达式不重复承载这两项）。另提供 #{组员实发提成}、#{组员平均票数} 用于组维度门槛命中，聚合定义见下表。

表达式板块：条件表达式（系统）与计算公式（系统）

与页面「表达式与计算公式编辑器」两栏、工具条占位符及灰字说明一致；落库字段名、函数白名单以规则引擎注册表为准，下表为 MVP 原型约定。

职责划分：条件表达式产出命中判断（布尔/可判定真值），与上方结构化匹配区语义须可校验、可对账；计算公式仅在命中后参与应发金额调整，不参与是否命中。

计算公式（MVP 暂行）：工具条仅提供 #{应发提成}、#{客服提成扣}、#{表外成本扣}、#{绩效系数} 四项占位符。应发提成由引擎在命中后按「基数方案 + 毛利阶梯/计费重/固定比例 + 基本提成点（及累计规则等）」先算出主应发，本式主要承担绩效系数折扣与提成扣类加减；#{净利润基数}、#{基本提成点}、上月提成结转等暂不放入本工具条（由基数层或引擎步骤处理，与试算日志对齐）。组员实发提成、组员平均票数仅出现在条件表达式工具条，不进入计算公式四项。

#{应发提成} 应发提成 引擎在规则命中后，已按本页基数方案 + 计算方式（阶梯/计费重/固定比例）+ 基本提成点与累计规则算出的主应发金额（未乘绩效、未扣提成扣前快照，以实现字典为准）。 公式主项；典型写法与 #{绩效系数} 相乘后再减各项扣。
#{客服提成扣} 客服提成扣 同包客服侧先跑批结果（见下「三、提成扣说明」）。 应发链上扣减；未启用按 0 或引擎默认。
#{表外成本扣} 表外成本扣 月度调整 / 表外导入中大类为「提成扣」的当期汇总；与《域名说明》中 #{表外导入扣} 可映射为同一引擎字段（以实现注册名为准）。 应发链上扣减。
#{绩效系数} 绩效系数 月度调整 → 导入绩效系数（员工 + 账期），如 108%→1.08。 对应发主项做乘数折扣；未导入时默认 1.0 或阻断。

提供：+ - * / ( ) max( min( 及数字、, %。其中 % 若引擎定义为「取模」而非「百分比字面」，则须禁止与业务混用或在保存校验中拦截；百分比录入仍在「固定比例」等结构化区完成。

三、提成扣说明（业务口径）

与业务域名说明文档「三、提成扣说明」一致；本页「提成扣配置」与公式中的 #{客服提成扣}、#{表外成本扣}（与 #{表外导入扣} 可同源映射）对应。

  • 1. 客服提成：当期所属规则包若存在需计提/需扣的客服提成口径，引擎须先完成客服侧计提，得到当期可确认的客服提成结果，再作为「客服提成扣」参与本条规则（及同包内其他主体角色）之后的应发扣减链。即有客服提成则先算客服，再算当前规则主体；与《域名说明》中「业务员先跑、主管后跑」等顺序原则一并体现在跑批/试算步骤中。未命中客服或未配置时该项按 0 或引擎默认（与试算日志一致）。
  • 2. 表外导入（表外成本扣）：仅汇总当期在表外导入/月度调整中入账、且记录类型为「提成扣」的表外成本金额；公式区按钮文案为表外成本扣，占位符为 #{表外成本扣}（与历史文档 #{表外导入扣} 可映射同一字段，以实现注册名为准）参与扣减；非「提成扣」类型的导入不得计入本扣减项（可进入毛利扣等其他口径，见月度调整表 PRD）。
  • 本页「是否启用提成扣」「提成扣类型」：MVP 仅枚举两种——客服提成、表外导入（与《域名说明》「三、提成扣说明」一致）。总开关关闭则整条扣减链不执行；开启时仅对已勾选类型按上款口径取数并代入「计算公式」中 #{客服提成扣} / #{表外成本扣}。

二期《问题清单》产品回复摘录（规则 · 2026-05-11）

依据《提成管理系统-问题清单-产品回复开发.docx》与开发 Q&A 对齐；若与业务签字版冲突以签字版为准。

  • Q2 计算方式三 Tab：MVP 一条规则仅一种计算方式（三选一）。服务端只持久化当前激活 Tab；非激活 Tab 不作为引擎输入。保存/发布只提交激活 Tab；切换 Tab 可提示「仅当前页签会保存」；可选离开时清空未激活 Tab（非强制）。
  • Q3 结构化 vs 表达式：MVP 以结构化匹配区为权威；表达式由保存时按结构化生成/覆盖，或用户编辑时在保存前做一致性校验，不一致则阻断。引擎只消费通过校验的最终表达式快照。
  • Q4 保存草稿：草稿不替代已发布版本；已发布继续对外生效直至「保存并发布」。试算默认绑定当前生效发布版；「按草稿试算」须显式选草稿版本（可二期）。
  • Q5 匹配优先级：数值越大越优先；仅同一规则包、同一互斥/编排上下文内排序。同优先级：MVP 建议阻断保存，或按规则编码字典序（二选一并写死）。
  • Q6 累计规则（建议默认，待财务签字）：累计加成基数为基本提成点；年限自入职日期起算（满整年 +2%，不足年不计）；「累计上限」指最终提成比例封顶。
