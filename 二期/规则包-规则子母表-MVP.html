MVP 说明：
1）一个规则包可包含多条规则，本页展示包与包内规则的对应关系；列表上方工具条放置跳转规则编辑页、新增规则包与规则、基数维护；基数方案编辑与清单导出在「基数维护」内闭环，本列表不提供。
2）勾选首列「规则明细」复选框后，在该行下方展开包内规则，可多行同时展开；点击规则包名称或编码进入「规则包详情」只读页（支持 ?code= 直达）。
3）当前明细为包内规则副本视角，便于版本隔离与追溯；如未来做规则复用，可增加「来源规则 ID」列。
4）销售产品、基数方案、提成扣、回款率等口径以各条「编辑规则」为准，本页仅承载包级编排与角色关系。 5）包级组织：分公司必填；提成角色必填；部门、小组选填，与「人员映射表」维度一致，便于规则命中与列表筛选。
6）联调时建议增加校验：同包内规则优先级冲突、互斥组冲突、命中条件重叠。
7）历史版本：MVP 不在本列表工具条提供入口；进入「规则包详情」→「版本历史」查看时间轴与对比（与主路径《规则包与包内规则》PRD 一致）；试算/归档仍记录当时快照版本号。

规则包与包内规则列表（MVP · 二期壳）

本页聚合规则包主档与包内规则只读视图，章节结构与主路径《规则包与包内规则》PRD 一致；本 HTML 位于二期/，详情与部分工具条使用 ../ 指向仓库根目录原型。单条规则口径见《编辑规则》；包主档编辑见《新增编辑规则包》；基数见《基数维护》。

筛选区（本页首卡）

  • 规则包名称/编码：关键字模糊匹配。
  • 业务板块：与规则包客户类型一致，MVP 枚举仅直客、同行（加「全部」不按板块过滤）。
  • 状态：数据字典（生效中 / 草稿 / 停用的展示以主档为准）。
  • 生效月份：与规则包生效区间求交集；主档无起止时按产品默认（如长期有效）参与交集判断。
  • 分公司 / 部门 / 小组：模糊筛；筛选区留空表示不按该维过滤（不限维），避免与列表行「—」空值主档混淆。

列表工具条

  • 本文件为二期壳：规则编辑页 / 新增规则包 / 基数维护等按钮使用 ../ 指向根目录同名页。
  • 规则编辑页 / 新增规则：跳转规则中心编辑态。
  • 新增规则包：打开规则包编辑弹窗。
  • 基数维护：跳转基数维护；新建/编辑基数方案、导出清单等在基数模块闭环，本列表不提供。
  • 本列表不提供：规则引擎中心入口、列表导出、历史版本按钮等（与页脚 MVP 说明一致）。

规则包列表（主表）

  • 规则明细：首列勾选在同包下一行展开包内规则子表（可多包同时展开）。
  • 规则包名称 / 编码：点击进入「规则包详情」只读页（../规则包详情-MVP.html?code=）。
  • 分公司 * ：主档必填组织维；页眉强调提成角色亦为包主档必填（与《新增编辑规则包》一致）；部门 / 小组选填，未填列表展示「—」。
  • 包含规则数：当前快照下关联条数（是否计入草稿以实现为准）。
  • 版本：当前对外已发布展示版本；完整轨迹在详情页「版本历史」。
  • 状态：主档当前状态标签。

规则明细（展开区）

  • 展示该包下规则只读副本：名称/编码、适用角色、服务场景、计算方式、优先级、编排组、状态及操作链至「编辑规则」（路径以实现为准）。
  • 与「月度调整表」运单明细勾选展开交互一致；非实时编辑，修改须走编辑页或包编辑。
  • 销售产品、基数方案、提成扣、回款率参与口径以各条编辑规则为准，本展开区不承载公式与基数定义（详见《编辑规则》PRD）。

包内编排与角色策略（与规则层边界）

  • 同一规则包内：同销售产品/角色可相加或按互斥/优先级顺序执行，以主档编排策略与规则层标记为准。
  • 不同角色默认链路：通常先业务员后主管/经理；客服角色若配置优先则客服子链可先跑（与规则中心链路一致）。
  • 规则包主档负责命中范围与包级编排；规则层负责公式、扣减链、客户/场景条件等明细口径。
  • 不跨包复用：规则包与规则不可直接引用他包规则；共用能力优先复用基数口径。

命中链、排他性与复用（16 · 17 · 22）

  • 账期命中（16）：试算须先人员映射 → 再规则包 → 最后规则；三者对同一账期同时有效方可进入计提。
  • 规则包唯一（17）：同一提成角色 + 分公司 + 部门 + 小组在同一时间片内不得多个「生效中」规则包并存；保存/发布前须校验。
  • 试算以账期为主线；若版本切换导致同账期、同组织维多包并存，视为冲突数据，须系统提示并人工消解。
  • 单条规则生效区间须保留；入职时间等仅作人员映射或规则附加条件，不宜替代规则生效边界。
  • 复用（22）：基数方案可复用；规则包与规则不可复用（编码全局唯一，复制须新码）。

版本与历史

  • 入口：规则包详情 →「版本历史」标签（列表无独立「历史版本」按钮）；行内「历史」若实现，与详情同一抽屉并预选包。
  • 时间轴：版本号、发布/保存时间、操作人、状态变更、主档 Diff、包内规则编码快照。
  • 对比：两版本包内规则清单只读对比；纠错走「另存为新版本」。试算/归档落 规则包版本ID 便于审计。

字段有/无时的匹配（本页 · 列表与筛选）

  • 筛选区部门/小组为空：不按该维过滤（不限）。
  • 列表行部门/小组为「—」：主档未填；与人员映射匹配须与映射侧同维同时为空对齐，否则遵循人员映射 PRD 细粒度优先 / 不回退（细则见主路径 PRD）。
  • 生效月份：与包生效区间交集逻辑同筛选区说明。
  • 版本列：当前已发布版本号；完整轨迹见详情「版本历史」。

字段取值来源（本页）

  • 规则包名称/编码、组织维、状态、版本号：规则包主档 + 发布流水；列表为当前主数据视图。
  • 包含规则数：包—规则关联按包聚合（是否含草稿以实现为准）。
  • 展开区规则行：当前快照下包内子集，与「编辑规则」可读副本一致。
  • 筛选枚举：业务板块受客户类型约束（直客/同行）；其它枚举来自数据字典。
