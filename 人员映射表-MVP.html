人员映射表 / 弹窗 PRD（开发测试版）

二期标题前带角标的小节为与问题清单 16～22 等对齐的补充说明。

背景与目标

统一维护“员工工号、员工昵称、员工姓名、提成角色、时间区间”，为规则命中、试算、归档提供稳定主键与可追溯口径，避免昵称/姓名变化导致归因漂移。

用户与场景

  • 财务：校验某员工某月是否满足发放条件与时间边界。
  • 测试：覆盖同工号多记录、时间重叠、规则包命中失败场景。
  • 人事/销售支持：转岗、离职后账号继承、修改指定规则包等需落多段映射；首次批量导入可不覆盖本逻辑，以编辑/单条新增为主路径。

字段定义与口径

  • 唯一主键：员工工号（业务上由员工昵称 + 员工姓名与员工主数据匹配后落点；见下「导入 / 编辑」校验差异）。
  • 批量导入：以模板中的员工昵称、员工姓名与员工管理/主数据匹配并解析工号后入库；匹配不上则预检失败，需修正模板或先维护主数据。
  • 编辑保存：仅对本次提交中变更的新值做与主数据/业务规则相关的校验；本条记录已落库的历史字段（如入库时的昵称、姓名等快照）不再与当前员工管理数据做回溯比对——员工管理侧若已修改花名册，历史上传数据与现网不一致为预期情况，不因「旧数据对不上现网」而拦截保存。
  • 二期优化 结算提成时间：用于业绩/提成起算参考日期；默认等于入职时间，允许在新增/编辑/导入中单列维护（晚于入职的场景单独标记）。与「映射生效」「提成计算」区间并行存储；命中规则以产品口径为准。
  • 必填：分公司、提成角色、映射起始、人员状态。
  • 区间口径：默认“映射区间 = 提成区间”；若单独填写提成区间，必须被映射区间完全包含。
  • 映射生效状态（列表列）：仅区分 生效中 与 已失效。当前业务日期落在映射生效起始～截止（截止为「长期」则视为未到期）内为生效中；否则为已失效。不提供「即将到期」预警态；到期前提醒通过调度/报表或运维配置另行处理。
  • 提成计算状态（列表列，对应计提是否在区间内；业务上亦称提成发放口径）：仅区分 计提中 与 已失效。当前业务日期落在提成计算起始～截止内、且满足计提前置条件为计提中；区间外或停止计提/不发提成等等价场景展示为已失效（细分原因可走备注与后台枚举）。不提供「即将到期」预警态。
  • 可发提成时间：不可超过映射生效区间上限。
  • 人员映射关联规则：按分公司 + 提成角色 + 生效时间（账期）匹配规则包；若人员映射同时存在分公司 + 部门 + 小组，则优先按分公司 + 部门 + 小组 + 提成角色 + 生效时间（账期）匹配规则包。若命中多个规则包，则关联结果置空；若未命中，则关联结果置空。

交互说明

  • 列表分 Tab：
    • 全部映射：全部数据
    • 角色异常：匹配角色为空
    • 已失效：映射时间已过期
    • 未关联规则：关联规则为空
  • 新增：点击弹出弹窗，填写新数据。若同一员工工号在系统中已存在映射，则校验当前映射时间是否与已有数据交叉；存在交叉时提示：是否生成新的周期。
  • 编辑：列表「编辑」打开独立页或弹窗；保存逻辑含同工号区间校验。同工号、映射生效与提成计算区间均与入库一致且仅提成角色变更时，展示选项：勾选「生成新的映射记录」则新增一条；不勾选则覆盖本条。
  • 列表不展示行内操作列，编辑、批量导入、新增在列表右上方工具栏。
  • 批量导入均以弹窗页打开并可右上角关闭。
  • 批量导入支持预检：预检结果分通过/失败等状态（如全部通过、存在失败、全部失败）；失败行在导入页列表展示。

主管小组成员（主管 / 经理）

  • 展示条件：仅在「全部映射」Tab 下勾选单行，且该行提成角色为「直客主管」「同行经理」「客服中心主管」之一时，在表格下方展开「主管小组成员」区块；未勾选、多选或非主管/经理时展示简要提示文案。
  • 默认带出规则：成员候选池为映射表中提成角色名称含「业务员」或「客服」的人员（可与分公司、部门、汇报关系等维度裁剪，以实现为准）；默认列表可按组织归属预填或由运营初始化。
  • 成员明细列：除员工与组织、提成角色外，展示客户名称与对应客服（归因/服务关系示意，以实现口径为准）；不包含「调整类型」类字段（与月度调整运单明细列对齐）。
  • 列表区（只读）：展示当前已保存的成员（工号、昵称、姓名、部门、提成角色、客户名称、对应客服），与上方勾选的主管联动展示。
  • 编辑交互：点击「编辑小组成员」打开弹窗——左侧为候选池，右侧为当前小组成员；勾选行后使用「添加选中 →」「← 移除选中」在两侧移动；在弹窗内点保存后更新列表区并关闭弹窗；取消、关闭遮罩或 Esc 放弃本次弹窗内修改。

账号映射多记录与关键信息变更

适用于同一系统账号多条映射、转岗/离职后账号延续、修改引用规则包等；首次全量导入可不套用本节时间分段策略，以导入模板与预检为准。

  • 同工号同映射区间、仅提成角色变更（编辑页）：当员工工号、昵称、姓名与入库一致，且映射生效区间与提成计算区间均未改动，仅提成角色修改时，提供「生成新的映射记录」勾选：勾选则新增一条映射（同一员工多角色命中）；不勾选则覆盖本条。与「区间变更生成新段」的触发条件不同，以实现字段比对为准。
  • 关键信息变更 + 生效时间：编辑时若修改了关键信息（如分公司、提成角色、映射/提成区间、指定规则包等，具体字段以保存校验配置为准），且新的生效时间落在既有映射记录的生效区间之外（与原区间无交叉、或产品定义为需独立成段的新时段），则生成一条新的映射记录，历史记录保留可追溯；是否自动截断上一条的截止时间以实现为准。
  • 新增（单条）：录入的员工昵称 + 员工姓名须与员工主数据匹配工号；匹配规则与导入一致。若与当前工号/账号意图不一致：工号一致且新生效时间落在既有区间之外需新段落的，生成新记录；工号不一致或无法匹配主数据：阻断保存，提示「请使用新增映射」完成新人入库，禁止错绑。
  • 编辑：与上条「字段定义」一致——不强制校验历史快照与现网主数据是否仍一致；仅对本次修改涉及字段做必要校验（如修改了可编辑的姓名等，则只校验新值与工号/主数据关系）。工号、昵称在编辑页只读，换人/换号走新增映射。

二期试算命中链与账期（16 · 17 · 22）

  • 账期独立版本：人员映射、规则包、规则均以生效区间 / 账期独立版本参与命中；试算引擎顺序为人员映射 → 规则包 → 规则，须同一账期三者同时有效才进入计提链路。
  • 规则包唯一性：同一提成角色 + 分公司 + 部门 + 小组在同一时间片内不得存在多个「生效中」规则包；与下节「多命中置空」互补：多包命中与维度过宽命中均须人工消解。
  • 复用边界：基数方案可复用；规则包与规则不可复用（编码唯一，复制生成新编码）。

与其他模块联动逻辑

  • 规则包匹配主线：按分公司 + 提成角色 + 生效时间（账期）匹配规则包；人员映射中的分公司、提成角色与当期账期需同时落在有效区间内。
  • 组织维度优先级：若人员映射同时存在分公司 + 部门 + 小组，则优先按分公司 + 部门 + 小组 + 提成角色 + 生效时间（账期）去匹配规则包；若该更细粒度维度未命中，再视产品策略决定是否回退到仅按分公司 + 提成角色 + 生效时间（账期）匹配。当前原型默认不自动回退，避免跨组织误命中。
  • 多命中处理：同一条映射在同一账期内若命中多个规则包，则关联结果置空，并记录为冲突数据，供人工处理。
  • 未命中处理：若无任何规则包命中，则关联结果置空，进入未关联规则页签。
  • 月度调整：按员工工号+月度关联绩效系数与表外调整。
  • 月底归档：取归档批次账期内有效映射记录作为归因依据。

二期规则包历史版本查看

  • 本表不维护规则包版本时间轴；员工当前引用的规则包编码与生效区间见本列表及「引用规则包」弹窗。
  • 需查某包历史发布记录：到规则包与包内规则列表 →「历史版本」（与《规则包列表》PRD 一致）。

  • 部门/小组为空：匹配规则包时仅携带分公司 + 提成角色 + 账期；若映射与包两侧「部门/小组」一侧有一侧非空，按已选策略细粒度优先或不回退（见联动逻辑节）。
  • 引用规则包为空：进入「未关联规则」视图；不参与试算命中链第二步。
  • 提成计算区间未单独填写：默认与映射生效区间一致（须被映射区间包含）。
  • 结算提成时间（若有）为空：默认等于入职时间（二期列）。

二期字段取值来源（本页）

  • 员工工号、昵称、姓名：员工主数据匹配结果；列表可为历史快照。
  • 分公司、部门、小组、提成角色、区间、人员状态：本页维护 / 导入模板。
  • 关联规则包编码：引用规则包保存结果或手工选择已发布包。
  • 主管小组成员：主管行保存的组员关联 + 映射表候选人池裁剪。

非功能性要求
