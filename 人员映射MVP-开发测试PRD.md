## 1. 需求背景

### 1.1 业务背景

- 提成按**人 + 月汇总**计算，需将订单/子单上的业务员、客服等与**系统账号、员工、提成岗位**稳定对应。  
- 主数据侧可能**无独立岗位字段**，仅有部门与负责人标记；**提成岗位（含直客主管等）以人工在映射表认定为准**。  
- 需支持**时间轴**：映射生效区间、财务口径的提成计算区间、人事入/转/离日期可并存。  
- 规则比例、互斥组等在**规则中心**配置；本模块不维护规则逻辑，仅维护「谁以什么身份参与归因」。

### 1.2 建设目标（MVP）

- 提供人员映射**列表检索**、**单条新增/编辑**、**批量导入**、**模板下载与导出**（导出以原型为准，可二期）。  
- 列表折叠展示部分人事信息；完整字段在新增/编辑页维护。  
- 为后续订单归因、试算提供**可查询、可审计**的映射数据。

### 1.3 非目标（本期可不实现）

- 列表/表单中**服务场景（service_mode）**列（原型脚注：可二期与规则中心对齐后增加）。  
- 与规则引擎的**自动规则匹配**（指定规则字段为预留，见下文）。  
- `employee_id` 在 UI 上展示（建议后端存储；导入模板可要求）。

## 3. 人员映射列表页

**文件**：`personnel-mapping-list-mvp.html`

### 3.1 页眉区域

| 元素 | 类型 | 说明 |
|------|------|------|
| 标题 | 文案 | 「人员映射表」 |
| 描述 | 文案 | 说明 MVP 职责：账号—员工—部门/小组—提成岗位、映射生效与提成计算区间、发放状态；归因以映射生效为准；规则在规则中心。 |
| 下载模板 | 按钮 | 触发下载导入用 Excel 模板（文件版本号建议后端配置）。 |
| 导出 | 按钮 | 按**当前筛选条件**导出列表数据（格式 xlsx/csv 由实现定，需与测试约定）。 |
| + 新增映射 | 链接（样式同主按钮） | 跳转 `personnel-mapping-add-mvp-page.html`。 |

**逻辑说明**

- 「下载模板」与批量导入页「下载模板」应使用**同一模板文件**或同一下载接口。  
- 「导出」无筛选条件时导出全量（或当前页，**必须产品二选一并写死**；建议默认导出筛选结果集）。

---

### 3.2 查询条件区（每一块都要测）

**区块标题**：「查询条件」

| # | 字段名称 | 控件类型 | 占位/选项 | 查询逻辑说明 |
|---|----------|----------|-------------|----------------|
| 1 | 系统账号名称 | 单行文本 | 模糊搜索 | **模糊匹配** `account_name`（或登录名）。空则不作为条件。 |
| 2 | 员工姓名 | 单行文本 | 模糊搜索 | **模糊匹配** `employee_name`。空则不作为条件。 |
| 3 | 所属部门 | 单行文本 | 模糊搜索 | **模糊匹配** `dept_name`（或部门路径）。空则不作为条件。 |
| 4 | 提成岗位 | 下拉 | 全部 + 8 个枚举 | 选项：直客业务员、直客主管、直客客服、同行客服、客服中心主管、同行业务员、同行经理、渠道客服。选「全部」或空：**不按岗位过滤**。 |
| 5 | 映射生效起始 ≥ | 日期 | — | 筛选「映射生效结束日期 ≥ 该日期 **或** 长期有效」且「映射生效起始 ≤ 查询区间逻辑」**（需后端统一区间语义）**。推荐语义：**存在一天**落在 `[起始, +∞)` 与映射区间有交集且映射起始 ≥ 所选日期时命中（与产品确认后写进接口文档）。MVP 可简化为：`mapping_effective_start >= 所选日期`。 |
| 6 | 映射生效截止 ≤ | 日期 | — | 与上成对使用；若只填一侧则单侧生效。推荐与产品确认：`mapping_effective_end <= 所选日期` 或 `end IS NULL` 表示长期时与「截止 ≤」的交互规则。 |
| 7 | 人员状态 | 下拉 | 全部、在职、离职 | 等于所选枚举；全部则不过滤。 |
| 8 | 提成发放状态 | 下拉 | 全部、正常、停止 | 等于所选；全部则不过滤。 |

**按钮**

| 按钮 | 行为 |
|------|------|
| 重置 | 清空所有查询控件为默认态（日期清空、下拉为「全部」、文本清空）。 |
| 查询 | 携带当前筛选参数请求列表接口，`pageNo` 重置为 1（建议）。 |

**测试要点**

- 两日期同时填写时边界包含关系；只填起始/只填截止；非法日期（若前端校验）。  
- 下拉「全部」与空文本组合。  
- 超长关键词、特殊字符、SQL 注入防护（后端参数化）。

### 3.3 列表表格 — 列定义

**表头分组（视觉）**：前四列为「基础」绿区；映射生效、提成为「时间」黄区；其余默认色。

| # | 列名 | 数据来源字段（建议） | 展示规则 |
|---|------|----------------------|----------|
| 1 | 系统账号名称 | `account_name` | 纯文本 |
| 2 | 员工姓名 | `employee_name` | 纯文本 |
| 3 | 所属部门 | `dept_name` | 纯文本 |
| 4 | 所属小组 | `group_name` | 可空，空显示「—」或空白（与产品统一） |
| 5 | 映射生效 | `mapping_effective_start` ~ `mapping_effective_end` | 主行：`YYYY-MM-DD ~ YYYY-MM-DD` 或 `~ 长期`（`end` 为空时）。**副行小字**：入职、转正；若有离职且与映射相关可展示「离职 YYYY-MM-DD」（原型第 2 行在映射列下展示离职）。 |
| 6 | 提成计算 | `commission_calc_start` ~ `commission_calc_end` | 同上格式。副行可为固定提示文案：「与映射可不同，以财务口径为准」（可选配置是否显示）。 |
| 7 | 提成岗位 | `commission_role` | 文本 |
| 8 | 发放状态 | `commission_pay_status` | 标签：正常（绿）、停止（灰） |
| 9 | 人员状态 | `employee_status` | 标签：在职（绿）、离职（红） |
| 10 | 指定规则（预留） | `assigned_rule_code` | 有值显示编码；无值显示「—」 |
| 11 | 操作 | — | 见 3.4 |

**排序与分页（原型展示）**

- 工具栏文案：`共 {total} 条，当前第 {pageNo} 页`（实现时需 `pageSize`）。  
- 分页：每页 20 条（原型）；上一页 / 页码 / 下一页。  
- **默认排序建议**：`updated_at DESC` 或 `mapping_effective_start DESC`（产品定）。

## 4. 批量导入页

**文件**：`personnel-mapping-import-mvp-page.html`

### 4.1 页眉

| 元素 | 说明 |
|------|------|
| 返回列表 | 同列表 URL。 |
| 标题 | 批量导入人员映射 |
| 说明 | 上传校验后确认写入、生成批次号。 |
| 下载模板 | 与列表页模板一致。 |

### 4.2 表单字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 导入批次名称 | 建议必填 | 用于审计与批次列表展示；默认可生成「人员映射导入_日期_序号」。 |
| 重复数据处理 | 必选 | 枚举三选一：**阻断并报错** / **覆盖同账号同岗位重叠区间** / **跳过重复行**；导入引擎按选项处理冲突行。 |

### 4.3 上传区

| 逻辑 | 说明 |
|------|------|
| 支持格式 | `.xlsx`（若需 xls 另议）。 |
| 行数限制 | 建议单次 ≤ 5000 行；超出前端提示或后端拒绝。 |
| 选择文件后 | 异步解析 → 返回校验结果统计 + 明细表；**未确认前不落库**。 |

### 4.4 校验结果统计（四个数字卡片）

| 指标 | 含义 |
|------|------|
| 总行数 | 解析得到的有效数据行数（不含表头）。 |
| 通过 | 校验通过可导入。 |
| 预警 | 可导入但需关注（如重叠将覆盖）；策略由产品定义是否允许在存在预警时提交。 |
| 失败 | 阻断行；存在失败且策略为「阻断」时，**禁止**「确认导入」。 |

### 4.5 校验预览表列

| 列 | 说明 |
|----|------|
| 行号 | Excel 行号，便于用户改表。 |
| 系统账号 | 导入列。 |
| 姓名 | 导入列。 |
| 提成岗位 | 导入列。 |
| 结果 | 通过 / 预警 / 失败（标签）。 |
| 说明 | 失败原因或预警原因文案。 |

### 4.6 底部操作

| 按钮 | 逻辑 |
|------|------|
| 取消 | 返回列表；若有未提交批次可提示「未保存是否离开」（实现建议）。 |
| 下载失败明细 | 仅失败行 > 0 时可点；生成带失败原因的文件。 |
| 确认导入 | 调用提交接口：`batch_id` + 文件或解析会话 id；**失败>0 且策略阻断则禁用或接口 400**。成功后跳转列表或结果页并提示批次号。 |

### 4.7 模板列与导入校验（建议后端清单）

**建议必填列**（与页脚 hint 一致）：系统账号名称、员工姓名、所属部门、所属小组（可空列）、生效起始、失效截止、提成计算开始/结束、人员状态、提成发放状态、提成岗位、指定规则、备注；日期格式统一。

**建议校验规则（测试用例来源）**

- 系统账号名称空 → 失败。  
- 必填时间格式错误 → 失败。  
- `生效起始 > 失效截止`（二者均非空）→ 失败。  
- 同账号同岗位区间重叠：按「重复数据处理」分支覆盖/跳过/阻断。  
- 与主数据姓名不一致仅预警（若启用 ID 列后则以 ID 为准）。

## 5. 新增人员映射页

**文件**：`personnel-mapping-add-mvp-page.html`

### 5.1 分区与字段 — 逐项说明

#### 5.1.1 基础信息（绿标题）

| 字段 | 必填 | 逻辑说明 |
|------|------|----------|
| 系统账号名称 | * | 与账号体系可检索；保存时建议校验账号存在性（若系统有账号服务）。 |
| 员工姓名 | * | 与 HR 主数据一致性：可预警不一致但不强制阻断（产品定）。 |
| 所属部门 | * | 文本快照或关联部门 ID（实现定）。 |
| 所属小组 | 否 | 可空。 |

#### 5.1.2 映射生效 & 提成计算区间（黄标题）

| 字段 | 必填 | 逻辑说明 |
|------|------|----------|
| 生效起始时间 | * | 映射开始生效日。 |
| 失效截止时间 | 否 | 空 = 长期有效（与产品约定截止日期上限二选一写清）。 |
| 提成计算开始时间 | 否 | 财务口径；空可表示与映射起始一致（需默认规则文档化）。 |
| 提成计算结束时间 | 否 | 同上。 |

**交叉校验**

- 若「失效截止」非空：`生效起始 <= 失效截止`。  
- 若提成计算起止均非空：`calc_start <= calc_end`。

#### 5.1.3 人事日期

| 字段 | 必填 | 逻辑说明 |
|------|------|----------|
| 入职时间 | 否 | 供规则条件、列表副行展示。 |
| 转正时间 | 否 | 同上。 |
| 离职时间 | 否 | 人员状态为离职时建议必填或预警。 |

#### 5.1.4 状态与提成岗位

| 字段 | 必填 | 逻辑说明 |
|------|------|----------|
| 人员状态 | * | 在职 / 离职。 |
| 提成发放状态 | * | 正常 / 停止。 |
| 提成岗位 | * | 下拉 8 项；**不允许空提交**。页面提示：主管/经理系统标记可能相同，**以本字段为提成身份准**。 |
| 指定规则（预留） | 否 | 有值时引擎可优先匹配该规则编码（二期）；MVP 可仅存库不参与计算。 |
| 备注 | 否 | 长文本。 |

### 5.2 底部操作

| 按钮 | 逻辑 |
|------|------|
| 取消 | 返回列表；有修改时离开确认（建议）。 |
| 保存 | 校验通过后 `POST` 创建；成功 toast + 跳转列表或停留编辑（产品定）。 |

### 5.3 与列表的一致性

- 列表展示的「映射生效」「提成计算」格式须与保存字段一致。  
- 列表副行展示的入职/转正/离职来自本表单字段。

## 6. 编辑人员映射页

**文件**：`personnel-mapping-edit-mvp-page.html`

### 6.1 入口与参数

- URL：`personnel-mapping-edit-mvp-page.html?id={mappingId}`  
- **生产逻辑**：`GET /api/.../personnel-mappings/{id}` 拉取详情填表。  
- **原型逻辑**：`id` 为 `1`/`2`/`3` 时填充演示数据（见脚本内 `demo` 对象）；缺省或非法 id 时回退为 `1` 或 404 页（实现定）。

### 6.2 展示字段

- 标题下 **映射 ID**：展示 `id` 或业务编号 `MAP-xxxxx`，便于客服报障。

### 6.3 表单区块

与**新增页字段完全一致**（四块：基础信息、映射与提成区间、人事日期、状态与岗位），区别仅在于：

| 差异点 | 说明 |
|--------|------|
| 数据加载 | 进入页时预填，不可把主键隐藏字段丢失（表单内放 `mapping_id` hidden）。 |
| 保存 | `PUT` 或 `PATCH` 更新同一 `id`；乐观锁可用 `version` 字段（可选）。 |
| 审计 | 记录修改前后差异（后端）。 |

### 6.4 底部操作

同新增：取消、保存。

## 9. 非功能需求（摘要）

- 列表接口在 10w 级数据下需分页索引合理（`account_name`、`dept_name`、`mapping_effective_start` 等）。  
- 导入解析异步 + 轮询或 WebSocket 进度（行数大时）。  
- 操作写审计日志。  
- 权限：按角色控制导入、编辑、停发（与总 PRD 权限表对齐）。

## 10. 测试用例清单（按页）

### 列表

1. 单条件 / 多条件组合查询结果正确。  
2. 重置后条件清空且重新查询为默认全集。  
3. 分页切换保持筛选条件。  
4. 编辑跳转 URL 带正确 id。  
5. 停发/恢复后状态与标签更新。

### 导入

6. 空文件、错误格式、超大行数。  
7. 三种种重复策略各执行一条用例。  
8. 存在失败行 + 阻断 → 确认导入不可用或接口拒绝。  
9. 成功导入后列表可见新数据、批次可追溯。

### 新增

10. 必填缺省提交 → 前端或 422。  
11. 日期交叉非法 → 拦截。  
12. 成功创建后列表可检索到。

### 编辑

13. 非法 id 展示错误页或空状态。  
14. 修改后保存再进入数据持久化。
