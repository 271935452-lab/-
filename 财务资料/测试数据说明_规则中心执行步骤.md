# 测试数据说明 · 逻辑加工表「人员映射」对齐 · 规则中心执行步骤

## ★ 全量生成（推荐：每人 1 条调整 + 1 条运单）

> **脚本**：`财务资料/generate_full_personnel_coverage.py`（或运行 `generate_mvp_test_data.py` → 内部调用同一逻辑）。  
> **数据源**：`财务资料/数据映射与逻辑加工表.xls` →「人员映射表」从 **第 3 行（示例「黑猫警长」）起到最后一行**，按 **系统账号名称非空** 逐行读取，共生成 **与映射表行数一致** 的数据。

| 输出文件 | 说明 |
|----------|------|
| **`测试数据_手工万能调整池_人员映射全量.xlsx`** | **每条映射一行对应一行**：`*系统账户名称`、`*员工姓名`、`*月度`、关联 **同一行的运单号** `WB-MAP202603-{映射表行号}`（行号为 Excel 0-based 行索引）；备注中含映射表行号、提成岗位、备注摘要。**费用性质**在行间轮换「扣毛利 / 扣提成」，以便枚举两类费用字典；金额按行号取确定性公式生成，并与对应运单的「表外毛利扣汇总 / 表外提成扣汇总」对齐。 |
| **`测试数据_运单底表_人员映射全量.xlsx`** | **每人一单**：运单号与上一文件联动；**业务员系统账号 / 客服系统账号** 根据 **`提成岗位`**（渠道客服 / 各客服岗 / 同行经理 / 直客业务员等）拆分命中路径（详见脚本 `pick_sales_cs`）；含计费重、整柜数、票数、规则编码演示列；**计算净利润** = 毛利 − 表外毛利扣汇总。 |

自检：**映射表行数** = **调整池数据行数** = **运单底表数据行数**（默认脚本打印人数）。

## 三、规则中心执行顺序（摘要）

1. 运单主数据 → 2. 行级毛利 → 3. 回款 → 4. **导入万能调整池** → 汇总表外毛利扣/提成扣 → 5. **人员映射**（系统账号 + 区间 + 提成岗位）→ 6. 规则包命中 → 7. 基数 → 8. 比例/阶梯 → 9. 分摊 → 10. 风险策略 → 11. 绩效等月度因子 → 12. 试算 → 13. 归档 → 14. 快照追溯。

**自检**：对每个运单号，将「手工万能调整池」中该运单 **费用性质=扣毛利** 的金额相加，应等于底表 **表外毛利扣汇总**。
