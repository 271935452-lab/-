引用规则包（独立弹窗）PRD

二期标题前带角标的小节为与问题清单 16～22 等对齐的补充说明。

背景与目标

在人员映射主数据已存在的前提下，为指定员工维护「引用哪一个已发布规则包」「从何时起生效」，使试算与归档能按分公司 + 提成角色 + 账期命中正确规则包。本页以独立 HTML交付，便于 window.open、新标签预览或与列表iframe 同屏嵌入，字段口径与人员映射内「引用规则包」能力对齐。

用户与场景

  • 人事 / 销售支持：员工调岗、换业务板块后切换规则包编码。
  • 财务：核对某员工某账期映射所绑定的规则包是否与政策一致。
  • 实施 / 开发：联调保存接口、校验与列表回填；测试覆盖生效区间边界与未选择规则包拦截。

字段定义与口径

  • 员工姓名（必填）：MVP 手填；实现侧可替换为搜索选人组件，选中后回填工号/部门。
  • 工号（必填）：与人员映射主键一致，保存时须能唯一定位或创建映射上下文（与列表「编辑引用」入口一致）。
  • 部门（选填）：展示组织路径，可与主数据联动只读。
  • 业务板块（必填）：直客 / 同行 / 渠道 / 客服；用于与规则包业务域及后续命中校验一致。
  • 规则包编码（必填）：下拉数据来自已发布规则包主数据；未选择时禁止保存。
  • 规则包名称（带出）（只读）：随编码联动展示，不参与提交主键。
  • 生效起始 / 生效截止：起始必填；截止为空表示长期有效。区间须落在人员映射有效策略内（与《人员映射表》PRD 区间包含关系一致）。
  • 状态：待生效 / 生效中 / 已失效；与映射生命周期及列表展示枚举对齐。
  • 变更说明（选填）：调岗、新政策切换、纠错等原因，写入审计备注。

交互说明

  • 关闭：右上角 ×、底部取消、点击遮罩（popup-root 空白处）、键盘 Esc；与全站一致：优先返回上一浏览页；若无可用历史则进入《提成试算表》试算步骤矩阵（开发参考）（有 opener 时仍优先尝试关闭弹窗）。
  • 重置：清空规则包选择与变更说明（其它字段是否一并清空以实现为准，原型仅清包与说明）。
  • 保存映射：提交后关闭或刷新列表由宿主页面决定；须做必填与区间校验。
  • 规则包下拉变更时同步更新「规则包名称」只读区。

二期与其他模块联动逻辑

  • 人员映射表：本页保存结果回写或新增「人员—规则包引用」记录；同一工号多段映射须满足区间不交叉或走产品定义的「生成新段」策略（见人员映射 PRD）。
  • 规则包主数据：仅可选择已发布包；草稿包不出现在下拉。
  • 试算 / 归档：命中顺序为人员映射 → 规则包 → 规则；三者同一账期同时有效才参与计算（与业务域名说明一致）。
  • 冲突：同一分公司 + 提成角色 + 账期若命中多个规则包，关联结果置空并需人工处理（与人员映射表 PRD 一致）。

二期规则包历史版本查看

  • 下拉中展示当前已发布包编码 + 当前版本号（只读带出名称）；包历史时间轴在规则包列表 → 历史版本查看。
  • 保存映射时可落规则包版本ID（若接口提供），便于与后续试算日志对账。

二期字段有/无时的匹配（本页）

  • 部门为空：不参与与规则包组织维度的额外收窄；命中仍以分公司 + 提成角色 + 账期为主（与人员映射主档一致）。
  • 生效截止为空：表示长期有效；须被人员映射区间包含。
  • 规则包未选：禁止保存。

二期字段取值来源（本页）

  • 员工姓名、工号、部门：人员映射上下文或主数据查询回填。
  • 业务板块、规则包编码/名称、生效区间、状态、变更说明：本页表单；规则包枚举来自规则包发布表。

非功能性要求

  • 权限：仅授权角色可修改他人映射；操作写审计（人、时、前后值）。
  • 幂等：重复提交同区间同包须可识别或更新策略明确。
  • 无障碍：对话框 role="dialog"、标题 aria-labelledby 保持。

本次范围与后续规划

  • 本次（MVP）：单页表单 + 右侧 PRD；规则包本地枚举；员工手填。
  • 后续 规则包远程搜索、员工联想搜索、与列表内嵌弹窗完全组件复用、未保存离开二次确认。
