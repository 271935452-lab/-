导入绩效系数弹窗 PRD

二期标题前带角标的小节为与问题清单模板对齐的补充说明。

背景与目标

按员工+月度维护绩效系数，用于月度提成汇总阶段调整。

用户与场景

  • 财务按月导入绩效。
  • 测试校验数值格式、主数据存在性与重复键。

字段定义与口径

  • 绩效系数必须为百分比转换后的数字系数（如 108% 对应 1.08）。
  • 判重键：员工 + 月份。
  • 同员工同月份命中历史记录：判定重复并写入失败明细。
  • 重复键处理由后台固定执行，不提供前端策略选择。

交互说明

  • 预检阶段给出结果。
  • 结果列表仅展示失败且需调整的数据行，不展示通过或预警。
  • 失败原因需明确到“格式错误/主数据/重复”等可定位类型。

与其他模块联动逻辑

  • 和人员映射按员工工号关联。
  • 和月度调整主表汇总后进入提成试算。

二期规则包历史版本查看

  • 绩效导入与规则包版本无绑定；对账依赖员工工号 + 账期月 + 批次号。

二期字段有/无时的匹配（本页）

  • 员工工号在主数据不存在：预检失败。
  • 月份格式非法：整行失败。

二期字段取值来源（本页）

  • 系数与员工、月份：导入文件。
  • 主数据存在性：员工主数据接口。

非功能性要求

  • 保证幂等导入与批次可回溯。
  • 支持失败明细下载复导。

本次范围与后续规划

  • 本次：手工模板导入。
  • 后续：与绩效系统 API 自动同步。
